Active Directory Security Hardening Checklist

Active Directory (AD) is the primary target for 90% of enterprise ransomware attacks. If an attacker gains Domain Admin privileges, the game is over. In 2026, securing AD requires moving beyond basic password policies to a comprehensive Tiered Administration model.

1. Implementing the Tiered Administration Model

The most critical step in AD hardening is preventing "Credential Theft." Tiered administration ensures that high-privilege credentials (like Domain Admins) are never exposed on lower-security workstations where an attacker might be lurking.

  • Tier 0: Domain Controllers, AD Forest, and Domain Admins. Highly restricted access.
  • Tier 1: Enterprise Servers and Applications.
  • Tier 2: User workstations and mobile devices.

2. The "Principle of Least Privilege" for Service Accounts

Service accounts are often the weakest link. They frequently have "Password Never Expires" set and excessive permissions. Hardening these involves:

  • gMSAs (Group Managed Service Accounts): Shift to gMSAs so Windows manages the password rotation automatically.
  • Disable Interactive Login: Service accounts should never be allowed to log in via RDP or locally at a console.

3. Securing against Kerberoasting and AS-REP Roasting

Attackers use these techniques to steal password hashes without interacting with a Domain Controller. To mitigate this:

# Use PowerShell to find accounts vulnerable to Kerberoasting
Get-ADUser -Filter {ServicePrincipalName -ne "$null"} -Properties ServicePrincipalName

Ensure all Service Principal Names (SPNs) are associated with accounts using long, complex passwords (25+ characters) or use gMSAs to render hashing attacks ineffective.

4. Cleaning Up Legacy Protocols

Legacy protocols are a goldmine for NTLM relay attacks. In 2026, your environment should be actively moving away from:

  • SMBv1: Disable it across the entire domain via Group Policy.
  • LLMNR & NetBIOS: Disable these to prevent "Responder" style man-in-the-middle attacks.
  • NTLM: Move toward Kerberos-only authentication using "Protected Users" security groups.

5. Monitoring for "Golden Ticket" Attacks

Once an attacker compromises the KRBTGT account, they can create a "Golden Ticket" that grants permanent access. Hardening requires rotating the KRBTGT account password twice a year (with a 24-hour gap between rotations) to invalidate any existing forged tickets.

Essential AD Checklist
  • Tiered Access: Admins never log into user workstations.
  • MFA: Required for all administrative logins and VPNs.
  • Audit Logs: Enable "Advanced Audit Policy Configuration" for Account Logon.
  • Orphaned Accounts: Automate the deletion of users inactive for >30 days.
  • Backups: Maintain offline, immutable backups of your NTDS.dit file.

Summary: Active Directory security is an ongoing battle of visibility and restriction. By implementing a Tiered Model and eliminating legacy protocols, you can force attackers to work much harder, giving your SOC team time to detect and respond to threats.
